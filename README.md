# AI-Powered Server Driven UI (SDUI) Banking Proof of Concept

This project demonstrates a production-quality Android application using Server Driven UI (SDUI) principles, designed to be driven by an MCP (Model Context Protocol) server and LLMs.

## Architecture
- **Core SDUI Engine**: Uses a "Widget" DSL to describe UI.
- **Dynamic Rendering**: Jetpack Compose based renderers registered via Hilt multibindings.
- **Visibility Engine**: Supports complex logical expressions (e.g., `is_premium == true && user_balance > 1000`).
- **Animations**: Integrated `AnimatedVisibility` and Lottie support.
- **Modules**:
    - `core-model`: Domain models and JSON serialization.
    - `core-renderer`: The central rendering engine and base widgets.
    - `feature-analytics`: Data visualization using Vico charts.
    - `core-network`: Ktor-based networking.
    - `navigation`: Dynamic routing.

## Tech Stack
- Kotlin 2.1.0, Compose, Material 3
- Hilt for DI
- Ktor for Networking
- Kotlinx Serialization
- Vico for Charts
- Lottie for Animations

## LLM Integration
The UI is intended to be generated by an LLM. Example prompt snippet:
`"Generate a banking home screen widget list in JSON format. Use 'balance_card' for the header and a 'lazy_column' of 'transaction_item' for the list."`

## Getting Started
1. Open in Android Studio Ladybug.
2. Sync Gradle.
3. Run the `app` module.
